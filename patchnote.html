Synchronous XHR Eliminated — Main Thread Unblocked
The original code fetched data/version.txt using XMLHttpRequest with the third argument set to false — a synchronous call on the main thread. This is browser-deprecated for exactly this reason: everything stops until the network responds. HTML parsing halts. JS execution halts. No layout happens. No pixels paint. On a slow 3G connection, this could produce a white-screen freeze of 500ms or more before users see anything. The call was replaced entirely with async/await fetch(), running the request off the main thread and completely unblocking first paint.

document.write() Eliminated — Chunks Load in Parallel
Two document.write() calls injected <script> tags for the results index and all data chunks. This API forces the HTML parser to stop mid-parse, destroys the browser's speculative preload scanner (which can no longer look ahead to queue other resources), and makes every chunk load one after another in a serial waterfall. It was replaced with dynamic document.createElement('script') injection and a Promise.all() that fires all chunk requests simultaneously — converting an N-step serial waterfall into a single-step parallel burst. The total load time drops from the sum of all chunk times to the time of the slowest single chunk.
Before — serial, parser-blocking
// Each call halts HTML parsing: document.write( '<script src="results_index.js">' ); chunks.forEach(c => { // One by one, every chunk: document.write( '<script src="data/'+c.file+'">' ); }); // Total time = SUM of all chunk fetches
After — parallel, non-blocking
// Index loads first (once): await loadScript('results_index.js?v='+ver); // Then ALL chunks fire simultaneously: await Promise.all( chunks.map(c => loadScript('data/'+c.file+'?v='+ver) ) ); // Total time = MAX of all chunk fetches
⚡ O(sum) → O(max) load time
✅ HTML parser kept running
📦 CSP adoption unblocked

Chart.js — defer Added to ~230 KB Script Tag
Chart.js was loaded in <head> with no attributes. A synchronous head script must be fully fetched, parsed, and executed before the browser can continue parsing any of the HTML that comes after it — this includes all the app UI structure. At ~230 KB minified, this is a significant parse cost. Adding the single word defer tells the browser to download it in parallel while HTML parsing continues, then execute it only after the document is fully parsed. One word. Entire render block gone.

The original head had a preconnect hint for fonts.googleapis.com only — the CSS API endpoint. Noise Overlay Promoted to Isolated GPU Layer
The full-viewport SVG noise texture in body::before was position: fixed but had no compositing hints. On every scroll event or CSS animation anywhere on the page, the browser's layer reconciler checks whether this fixed element — which covers the entire viewport — needs repainting. Sidebar Nav — All 6 <div onclick> Converted to <a href>
Every sidebar navigation item was a <div onclick="location.hash='...'">. This is the most fundamental keyboard accessibility failure: divs have no tab stop, no keyboard activation, no ARIA role, and are announced as nothing by screen readers. transition: all Replaced with Explicit Properties — 12 Rules
transition: all tells the browser to watch every single animatable CSS property for changes on every compositing frame: width, height, padding, margin, opacity, transform, color, background, border, font-size, and dozens more — even ones that will never change on these elements. On hover-state elements, this means the compositor evaluates the entire computed style diff on every pointer event. Each rule was updated to list only the two or three properties that actually transition, reducing the compositor's work to exactly what's needed. Twelve rules across nine selectors were updated.

Medium
font-display: swap Added — Eliminates FOIT on Google Fonts
Without font-display: swap, browsers use the block strategy: text renders invisibly (FOIT — Flash of Invisible Text) for up to 3 seconds while all three custom font families load from Google's CDN. Only after the timeout does a fallback system font appear. With swap, the browser renders immediately with the closest available fallback font, then swaps in the custom fonts when they arrive. The &display=swap&font-display=swap parameter was added to the Google Fonts URL — this tells Google's CSS generator to include the correct @font-face font-display: swap descriptor in the returned stylesheet.
📖 Text readable the instant the page loads
🚫 FOIT (invisible text period) eliminated

--topbar-h Design Token + will-change GPU Hint
Two complementary CSS improvements: the --topbar-h variable establishes a proper design token for a dimension that multiple elements depend on — any element that needs to offset by the topbar height can now reference the same variable rather than duplicating a magic number. The will-change: opacity addition to the noise overlay is a signal to the browser's compositing engine to pre-allocate a layer for this element before any animation or scroll event forces a reactive layer promotion, reducing the chance of a brief jank on first interaction.

Full Async Bootstrap Refactor — Single Entry Point Replacing 4 Disconnected Blocks
The original data loading was spread across three separate script blocks in <head> plus a DOMContentLoaded listener at the very bottom of the file. The sequence was: (1) sync XHR for version, (2) document.write for index script, (3) document.write loop for chunks, (4) wait for DOM ready, (5) call loadData. No error handling existed at any step. The entire sequence was consolidated into a single async bootstrap() function: fetch version → load index → load all chunks in parallel → merge → call loadData() when DOM is ready. The old bottom-of-file listener was removed as redundant.
Before — 4 disconnected blocks, no error handling
